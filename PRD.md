# PRD: 할 일 관리 앱 (Todo Manager)

## 1. 개요

매일 10~20개의 할 일을 관리하는 개인용 웹 앱.
브라우저에서 바로 실행되며, 별도 서버 없이 로컬에서 동작한다.

## 2. 목표

- 할 일의 추가, 수정, 삭제, 완료 처리를 간편하게 수행
- 카테고리별 분류로 할 일을 시각적으로 구분
- 진행률을 한눈에 파악
- 새로고침/브라우저 재시작 후에도 데이터 유지

## 3. 사용자

- 대상: 개인 사용자 1명
- 사용 환경: 데스크톱/모바일 브라우저
- 일일 할 일 수: 10~20개

## 4. 기술 스택

| 항목 | 선택 |
|------|------|
| 마크업 | HTML5 |
| 스타일 | CSS3 |
| 로직 | Vanilla JavaScript (ES6+) |
| 데이터 저장 | localStorage |
| 빌드 도구 | 없음 (정적 파일) |

## 5. 핵심 기능

### 5.1 할 일 추가
- 텍스트 입력 + 카테고리 선택 후 추가
- Enter 키 또는 추가 버튼으로 등록
- 빈 텍스트는 추가 불가

### 5.2 할 일 수정
- 할 일 텍스트를 클릭하면 인라인 편집 모드 진입
- Enter 키로 수정 확정, Esc 키로 취소
- 빈 텍스트로 수정 불가 (원래 값 유지)

### 5.3 할 일 삭제
- 삭제 버튼(x) 클릭 시 즉시 삭제

### 5.4 완료 체크
- 체크박스 토글로 완료/미완료 전환
- 완료된 항목은 취소선 + 투명도 처리

### 5.5 카테고리 분류
- 3개 카테고리: 업무, 개인, 공부
- 카테고리별 색상 구분 (좌측 보더 + 배경색 + 배지)

| 카테고리 | 키 | 색상 |
|----------|-----|------|
| 업무 | work | 파랑 (#2563eb) |
| 개인 | personal | 초록 (#16a34a) |
| 공부 | study | 보라 (#9333ea) |

### 5.6 진행률 표시
- 상단 프로그레스 바로 완료 비율 시각화
- "N / M 완료" 텍스트 표시

### 5.7 데이터 영속성
- 모든 변경 시 localStorage에 자동 저장
- 페이지 로드 시 localStorage에서 복원
- 저장 키: `todos`
- 저장 형식: JSON 배열

```json
[
  {
    "id": 1706500000000,
    "text": "보고서 작성",
    "category": "work",
    "completed": false
  }
]
```

### 5.8 필터링 기능
- 전체 / 미완료 / 완료 3개 필터 버튼 제공
- 선택된 필터에 따라 할 일 목록을 동적으로 필터링
- 프로그레스 바는 필터와 무관하게 전체 기준으로 표시

| 필터 | 키 | 표시 대상 |
|------|-----|-----------|
| 전체 | all | 모든 할 일 |
| 미완료 | active | completed === false |
| 완료 | completed | completed === true |

## 6. 데이터 모델

### Todo 항목

| 필드 | 타입 | 설명 |
|------|------|------|
| id | number | 생성 시각 타임스탬프 (Date.now()) |
| text | string | 할 일 내용 |
| category | string | "work" \| "personal" \| "study" |
| completed | boolean | 완료 여부 |

## 10. 구현 계획

### Phase 1: 데이터 계층 리팩토링
- DOM 직접 조작 방식에서 데이터 배열 기반으로 전환
- `todos` 배열을 중심으로 렌더링 함수 분리

### Phase 2: localStorage 연동
- 변경 시마다 `localStorage.setItem('todos', JSON.stringify(todos))` 호출
- 페이지 로드 시 `localStorage.getItem('todos')` 로 복원

### Phase 3: 수정 기능 추가
- 텍스트 클릭 시 input 으로 전환 (인라인 편집)
- Enter로 확정, Esc로 취소

## 11. 제약 사항

- 서버 없음: 모든 데이터는 브라우저 localStorage에 저장
- localStorage 용량 제한 (약 5MB): 일일 10~20개 수준에서는 문제 없음
- 단일 기기 사용: 기기 간 동기화 없음
- 외부 라이브러리 사용하지 않음
